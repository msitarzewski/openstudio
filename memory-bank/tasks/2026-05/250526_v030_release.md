## Patterns Applied

- **Variable fonts as single-file family** ‚Äî Google Fonts now ships Inter / JetBrains Mono / Space Grotesk as variable woff2 where the same file URL serves all weights. One `@font-face` per family with `font-weight: <min> <max>` covers every weight the design system uses, keeping the payload at ~100 KB total instead of nine separate files.
- **Subset by `unicode-range`** ‚Äî Latin range only (mirrors what Google Fonts serves for the `latin` subset). Browser still falls back to system fonts for non-Latin characters in user-typed names; matches behavior before self-hosting.
- **CSS-driven layout, minimal HTML restructure** ‚Äî Header layout change required only removing a wrapper div and converting `.header-right` to a flex container; the existing grid template absorbed the change with a single column-count tweak.

## Integration Points

- Static server MIME map (`server/lib/static-server.js:16`) gates which extensions are served ‚Äî without `.woff2` entry, fonts return false from `serveStatic` and fall through to 404.
- `body.broadcasting` CSS class on `<body>` still drives the visual broadcast-state cascade; header structural change doesn't affect it.

## Architectural Decisions

- **Decision**: Self-host fonts via single-file-per-family variable woff2 with latin subset
- **Rationale**: Matches the "no CDN dependencies for critical paths" rule in `projectRules.md`. Variable fonts collapse what would have been 9 files into 3. Latin-only keeps first-load ~100 KB; system fallback covers non-Latin user input.
- **Trade-off**: Diacritics in user-typed names (M√ºller, S√∏ren) render in system font instead of the brand font. Acceptable ‚Äî labels of the studio chrome are English-only, and same fallback behavior existed before for non-Latin scripts.
